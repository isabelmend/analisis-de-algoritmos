Introduccin

El anlisis y procesamiento de datos en dimensiones altas requiere tcnicas eficientes tanto en tiempo de cmputo como en uso de memoria. Una operacin fundamental en este contexto es el producto punto entre vectores, que constituye la base de numerosos algoritmos en aprendizaje automtico, recuperacin de informacin y anlisis de datos multivariantes. Tradicionalmente, este clculo se puede realizar mediante bucles explcitos, como en la funcin getmaxdot, que recorre todas las columnas de una matriz comparando cada producto punto (Malkov & Yashunin, 2020). Sin embargo, este enfoque resulta ineficiente cuando se manejan grandes volmenes de datos.

Para mejorar el rendimiento, se emplean tcnicas vectorizadas, que aprovechan operaciones matriciales optimizadas en libreras como NumPy o frameworks basados en BLAS/CUBLAS (Torres et.al., 2024). Estas versiones permiten calcular simultneamente todos los productos punto, reduciendo drsticamente el tiempo de ejecucin. Adems, cuando los tamaos de las matrices exceden la capacidad de memoria, se pueden implementar procesamientos por bloques, que dividen los datos en segmentos manejables, minimizando el uso de memoria sin sacrificar precisin.

En el contexto de estructuras de datos, el uso de estructuras elementales como arreglos, matrices y listas proporciona la base sobre la cual se construyen ndices ms sofisticados, como rboles de bsqueda, k-d trees o grafos HNSW, que permiten realizar bsquedas y comparaciones de vectores de manera eficiente sin tener que recorrer todos los elementos (Samet, 2006; Malkov & Yashunin, 2020). La correcta eleccin y manejo de estas estructuras es crucial para disear algoritmos escalables y optimizados, especialmente en aplicaciones de alto rendimiento o en anlisis de grandes volmenes de datos.

En este reporte se presentan y comparan diferentes enfoques para calcular el producto punto mximo entre vectores, evaluando tanto la eficiencia computacional como la memoria utilizada. Se implementan versiones clsicas, vectorizadas y por bloques, ilustrando cmo las decisiones sobre estructuras de datos y paralelizacin afectan directamente el rendimiento de los algoritmos.

  • Dado la matriz Q de tamao $d \times m$ y la matriz $X$ de tamao $d \times n$, para cada vector columna de $q \in Q$ encontrar el vector columna de $x \in X$ que maximiza su producto punto. Dicho de otra forma, el problema consiste en asociar cada $q$ con $\text{argmax}{q \cdot x x \in X}$. Este ser el problema de encontrar el producto punto mximo o top-1.

# Calcula el producto punto entre dos vectores 1D.
# Parmetros:
# u : array-like de dimensin (d,)
# x : array-like de dimensin (d,)
# Retorna s : El valor del producto punto u  x

def mydot(u, x):
  s = 0.0
  for ui, xi in zip(u, x):
    s += ui * xi
  return s

# Se define la funcin 'getmaxdot' para encontrar el vector columna en una matriz
# que produce el mximo producto punto con un vector dado

# Se define la funcin 'getmaxdot' donde:
# u: es el vector columna (d,)
# X: es la matriz de tamao (d, n)
# Regresa (ndice, valor) del mximo producto punto

def getmaxdot(u,X):
  maxpos = 0
  maxdot = mydot(u, X[:, 0])

  _, ncols = X.shape
  for i in range(1, ncols):
    d = mydot(u, X[:, i])
    if d > maxdot:
      maxpos = i
      maxdot = d
  return maxpos, maxdot

# Encuentra, para cada vector columna en Q, la columna de X que tiene
# el producto punto mximo, utilizando operaciones vectorizadas para eficiencia
# Parmetros: Q - Matriz cuyas columnas son los vectores de consulta.
# X - Matriz cuyas columnas se comparan con cada columna de Q.
# Retorna: max_indices : indices de las columnas de X que producen el producto
# punto mximo para cada vector columna de Q.

def top_maxdot(Q, X):
  # Q.T @ X produce una matriz de tamao (m, n) donde cada elemento (i,j)
  # es el producto punto entre la columna i de Q y la columna j de X
  dot_products = Q.T @ X
  max_indices = np.argmax(dot_products, axis=1)
  return max_indices

# Generacin del algoritmo en versin 'getmaxdot'

# Se retoma la versin vectorizada para encontrar el ndice de la columna con
# producto punto mximo
def top1_maxdot(Q, X):
    # Algoritmo vectorizado con producto matricial:
    dot_products = Q.T @ X
    max_indices = np.argmax(dot_products, axis=1)
    return max_indices

# Establecemos una versin por bloques para manejar matrices muy grandes y
# ahorrar memoria

# Parmetros: Q - vectores de consulta. X - Matriz a comparar
# block_size : Tamao de cada bloque de columnas de X para procesamiento
# Retorna: best_idx - ndices de las columnas de X con producto punto mximo
def top1_maxdot_blocked(Q, X, block_size=5000):
    m = Q.shape[1]
    n = X.shape[1]
    best_idx = np.zeros(m, dtype=int)
    best_val = np.full(m, -np.inf, dtype=np.float32)

    # Procesamos X en bloques
    for start in range(0, n, block_size):
      end = min(start + block_size, n)
      dots = Q.T @ X[:, start:end]
      local_idx = np.argmax(dots, axis=1)
      local_val = np.max(dots, axis=1)

      # Actualizamos solo donde encontramos un valor mayor al anterior
      mask = local_val > best_val
      best_val[mask] = local_val[mask]
      best_idx[mask] = start + local_idx[mask]

    return best_idx

def mydot(u, x):
  s = 0.0
  for ui, xi in zip(u, x):
    s += ui * xi

def getmaxdot(u, X):
  maxpos = 0
  maxdot = mydot(u, X[:, 0])
  _, ncols = X.shape
  for i in range(1, ncols):

# Se calculan los ndices de columna con producto punto mximo para cada vector
# columna de Q usando la getmaxdot
# Retorna: ndarray de ndices mximos por columna de Q
def getmaxdot_for_Q(Q, X):
  results = []
  for j in range(Q.shape[1]):
    results.append(getmaxdot(Q[:, j], X))
  return np.array(results)

Operaciones aritmticas $(\Sigma \text{ y } x)$:

  • Para cada producto punto: $2d$ operaciones ($d$ multiplicaciones + $d$ sumas).

  • En el algoritmo getmaxdot: $m \cdot n \cdot (2d)$ operaciones.

  • En la versin vectorizada: el producto matricial Q.T @ X requiere $ 2 \cdot m \cdot n \cdot d$ operaciones.

# Funcin que compara el rendimiento de diferentes mtodos para encontrar la
# columna de X que produce el producto punto mximo para cada columna de Q
# Parmetros: Q - Matriz de vectores de consulta, X - Matriz de vectores a
# comparar .
# Mtodo a usar: "vectorized" o "getmaxdot"

def comparison(Q, X, method="vectorized", block_threshold=2e8):
  d, m = Q.shape
  _, n = X.shape

  if method == "vectorized":
    # Si el tamao del problema supera el umbral, usamos la versin por bloques:
    if m * n > block_threshold:
      start = time.time()
      result = top1_maxdot_blocked(Q, X)
      elapsed = time.time() - start
    else:
      start = time.time()
      result = top1_maxdot(Q,X)
      elapsed = time.time() - start
    ops = 2 * m * n * d
  else:
      # Mtodo'getmaxdot' usando bucles
      start = time.time()
      result = getmaxdot_for_Q(Q, X)
      elapsed = time.time() - start
      ops = 2 * m * n * d

  # Estimacin de memoria utilizada por las matrices Q y X (en bytes)
  memory = Q.nbytes + X.nbytes

  return {
      "method": method,
      "m": m,
      "n": n,
      "ops": ops,
      "time_sec": elapsed,
      "memory_MB": memory / (1024**2)
  }

Los experimentos muestran una diferencia clara entre los dos algoritmos:

El mtodo $getmaxdot$ es simple y con bajo consumo de memoria, pero ineficiente en tiempo, incluso para tamaos moderados.

Por otra parte, el mtodo $vectorizado$ aprovecha la optimizacin de libreras de lgebra lineal, alcanzando tiempos y eficiencias varios rdenes de magnitud mejores. Aunque requiere ms memoria, gracias al procesamiento por bloques logra manejar problemas muy grandes.

Se puede decir que, en la prctica, solo el algoritmo $vectorizado$ es viable para aplicaciones reales a gran escala. (Ziogas et. al., 2021)

  • Qu costo es ms significativo?

Se observa que el tiempo de ejecucin es el costo dominante:

El conteo de operaciones en ambos mtodos es tericamente similar $(O(mnd))$, pero la forma de implementarlas cambia drsticamente el desempeo.

En $getmaxdot$, la ejecucin se vuelve prohibitiva a partir de $10^710^8$.

En el mtodo $vectorizado$, el tiempo crece mucho ms lentamente y se mantiene manejable incluso en $10^{10}$.

Por lo tanto, el tiempo real (y la eficiencia asociada) es el costo ms crtico al decidir qu algoritmo usar.

  • Relacin entre costo basado en conteo de operaciones y tiempo real

Ambos algoritmos requieren aproximadamente el mismo nmero de operaciones aritmticas (sumas y productos). Sin embargo, el tiempo real no depende solo de la cantidad de operaciones, sino de cmo se ejecutan:

En $getmaxdot$, las operaciones se realizan en un bucle secuencial en Python $$ baja eficiencia.

En $vectorizado$, las operaciones se delegan a rutinas optimizadas en C/Fortran con paralelizacin y SIMD $$ altsima eficiencia. (Scientific Python Community, 2022)

Esto explica por qu el conteo de operaciones predice la complejidad terica, pero no refleja el desempeo prctico si no se consideran optimizaciones de hardware y libreras.

  • Impacto de la memoria en cada algoritmo

Mtodo $getmaxdot$: muy bajo uso de memoria (procesa un vector a la vez). Es eficiente en este aspecto, pero sacrifica tiempo.

Mtodo $vectorizado$: uso de memoria mayor, debido a matrices intermedias. Sin embargo, con el enfoque de bloques el crecimiento se mantiene controlado (en MB, no en GB).

Se puede concluir que la memoria impacta al $vectorizado$, pero no llega a ser un factor limitante en los experimentos. En cambio, para $getmaxdot$ el problema no es la memoria, sino el tiempo.

  • Cmo podramos crear un mejor algoritmo?

  1. Bloques adaptativos: ya implementado, pero podra ajustarse dinmicamente segn el tamao de cach y RAM disponibles.

  2. Aproximacin en lugar de exactitud: en problemas de bsqueda top-k, se pueden usar algoritmos aproximados (ej. Locality Sensitive Hashing, Product Quantization) que reducen drsticamente tiempo y memoria sacrificando algo de exactitud (Jgou et.al., 2011).

  3. Paralelizacin explcita: aprovechar GPU o libreras multihilo (ej. CuBLAS en CUDA o PyTorch) para acelerar an ms el vectorizado (Torres et.al., 2024).

  4. Estructuras de datos especializadas: usar ndices (rboles de bsqueda, k-d trees, HNSW graphs) en lugar de comparar contra todas las columnas de X (Samet et.al., 2006).

Conclusin general

El anlisis confirma que la vectorizacin es la clave para resolver el problema top-1 de manera eficiente. El tiempo de ejecucin es el costo ms importante a considerar, mucho ms que la memoria. Aunque ambos algoritmos tienen la misma complejidad terica, en la prctica la eficiencia de las operaciones vectorizadas marca una diferencia de varios rdenes de magnitud. Para problemas an ms grandes, un mejor algoritmo debera combinar procesamiento por bloques, paralelizacin y, cuando sea aceptable, aproximaciones que reduzcan el nmero de operaciones exactas.
