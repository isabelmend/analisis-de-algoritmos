Introduccin

El ordenamiento de datos es una de las operaciones fundamentales en el mbito de la computacin, ya que constituye la base para optimizar numerosas tareas como la bsqueda, la organizacin y el anlisis eficiente de informacin. Los algoritmos de ordenamiento permiten reordenar los elementos de una lista o arreglo siguiendo un criterio especfico (por lo general, de menor a mayor), siendo esenciales en el diseo de sistemas eficientes para el manejo y procesamiento de datos a gran escala (Cormen et al., 2022).

Existen mltiples enfoques para realizar el ordenamiento, los cuales difieren en su complejidad temporal, uso de memoria y estabilidad. Entre los ms destacados se encuentran MergeSort, QuickSort y HeapSort, los cuales emplean estrategias de "divide and conquer" para alcanzar una complejidad promedio de $O(n \log{n})$. En contraste, algoritmos ms simples como BubbleSort presentan una complejidad cuadrtica $O(n^2)$, lo que los hace ineficientes para grandes volmenes de datos (Sedgewick & Wayne, 2011).

Adems, estructuras de datos avanzadas como la SkipList han surgido como alternativas eficientes para operaciones dinmicas de bsqueda y ordenamiento, combinando propiedades probabilsticas que permiten mantener un rendimiento promedio logartmico en inserciones, eliminaciones y bsquedas (Pugh, 1990).

El estudio comparativo de estos algoritmos resulta crucial para comprender sus fortalezas y limitaciones, as como para determinar cul es el ms adecuado segn las caractersticas del conjunto de datos y las restricciones del sistema. En este trabajo, se analizan cinco mtodos de ordenamiento: HeapSort, MergeSort, QuickSort, BubbleSort y SkipList, evaluando su desempeo en trminos de nmero de comparaciones y tiempo de ejecucin bajo distintos niveles de desorden en los datos.

class Counter:
    def __init__(self):
        self.count = 0

    # Funcin que realiza una comparacin entre 'a' y 'b' utilizando el operador
    # indicado. Incrementa el contador cada vez que se ejecuta una comparacin.
    def compare(self, a, b, op="<"):
        self.count += 1
        if op == "<":
            return a < b
        elif op == "<=":
            return a <= b
        elif op == ">":
            return a > b
        elif op == ">=":
            return a >= b
        elif op == "==":
            return a == b
        else:
            raise ValueError("Operador no vlido")

# Funcin 'heapsort': ordena el arreglo usando un heap binario
def heapsort(arr, counter):
    # Funcin auxiliar para mantener la propiedad de heap (max-heap):
    def heapify(arr, n, i):
        # Inicialmente el nodo actual es el ms grande:
        largest = i
        # ndices de los nodos hijos izquierdo y derecho:
        l, r = 2*i + 1, 2*i + 2
        # Compara con el hijo izquierdo:
        if l < n and counter.compare(arr[l], arr[largest], ">"):
            largest = l
        # Compara con el hijo derecho:
        if r < n and counter.compare(arr[r], arr[largest], ">"):
            largest = r
        # Si el nodo ms grande no es el actual, intercambia y heapifica recursivamente
        if largest != i:
            arr[i], arr[largest] = arr[largest], arr[i]
            heapify(arr, n, largest)

    n = len(arr)
    # Construye un max-heap:
    for i in range(n//2 - 1, -1, -1):
        heapify(arr, n, i)
    # Extrae elementos del heap uno por uno y los coloca al final:
    for i in range(n-1, 0, -1):
        arr[i], arr[0] = arr[0], arr[i]
        heapify(arr, i, 0)
    return arr

# Funcin 'mergesort': divide y conquista para ordenar el arreglo
def mergesort(arr, counter):
    if len(arr) <= 1:
        # Caso base: lista de un solo elemento:
        return arr
    mid = len(arr)//2
    # Ordena la mitad izquierda:
    L = mergesort(arr[:mid], counter)
    # Ordena la mitad derecha:
    R = mergesort(arr[mid:], counter)
    # Combina las dos mitades ordenadas:
    return merge(L, R, counter)

# Funcin 'merge' auxiliar para fusionar dos listas ordenadas:
def merge(L, R, counter):
    result = []
    i = j = 0
    # Compara elementos de ambas listas y agrega el menor:
    while i < len(L) and j < len(R):
        if counter.compare(L[i], R[j], "<="):
            result.append(L[i])
            i += 1
        else:
            result.append(R[j])
            j += 1
    # Agrega los elementos restantes si hay alguno:
    result.extend(L[i:])
    result.extend(R[j:])
    return result

# Funcin 'quicksort': divide y conquista usando un pivote
def quicksort(arr, counter):
    if len(arr) <= 1:
        # Caso base:
        return arr
    # Elegimos el pivote central:
    pivot = arr[len(arr)//2]
    left = [x for x in arr if counter.compare(x, pivot, "<")]
    mid = [x for x in arr if counter.compare(x, pivot, "==")]
    right = [x for x in arr if counter.compare(x, pivot, ">")]
    # Ordena recursivamente izquierda y derecha y concatena:
    return quicksort(left, counter) + mid + quicksort(right, counter)

# Funcin 'bubblesort': compara elementos adyacentes y los intercambia si estn
# desordenados
def bubblesort(arr, counter):
    n = len(arr)
    for i in range(n):
        for j in range(n-1-i):
            if counter.compare(arr[j], arr[j+1], ">"):
                # Intercambia si estn en el orden incorrecto:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

A continuacin, se implementa la estructura SkipList con el propsito de comparar el rendimiento de una estructura de datos probabilstica frente a los algoritmos de ordenamiento clsicos. A diferencia de mtodos como HeapSort o MergeSort, que aplican un proceso de ordenamiento sobre una lista completa, la SkipList mantiene los elementos ordenados de manera dinmica durante cada insercin, gracias a su arquitectura multinivel basada en saltos aleatorios.

Su inclusin permite observar cmo una estructura de datos puede mantener el orden de manera eficiente sin un algoritmo de ordenamiento explcito, con un costo promedio de bsqueda e insercin de $O(\log{n})$ y un comportamiento similar al de los rboles balanceados (Pugh, 1990). De esta forma, se demuestra que el orden puede lograrse tanto mediante procesos algortmicos como mediante la organizacin estructural de los datos.

# Clase que representa un nodo en la Skip List:
class SkipNode:
    def __init__(self, value, level):
        # Valor almacenado en el nodo:
        self.value = value
        # Lista de punteros 'forward' a los siguientes nodos en cada nivel:
        self.forward = [None] * (level+1)

# Clase que representa la Skip List:
class SkipList:
    def __init__(self, max_level=16, p=0.5):
        # Nivel mximo permitido en la lista:
        self.max_level = max_level
        # Probabilidad usada para decidir el nivel de un nodo:
        self.p = p
        # Nodo cabecera:
        self.header = SkipNode(None, max_level)
        # Nivel actual ms alto de la lista:
        self.level = 0
        # Para contar comparaciones:
        self.counter = None

    # Genera un nivel aleatorio para un nuevo nodo segn la probabilidad p:
    def random_level(self):
        lvl = 0
        while random.random() < self.p and lvl < self.max_level:
            lvl += 1
        return lvl

    # Inserta un valor en la Skip List:
    def insert(self, value):
        update = [None] * (self.max_level+1)
        current = self.header

        # Recorre los niveles de arriba hacia abajo para encontrar la posicin de
        # insercin:
        for i in range(self.level, -1, -1):
            while current.forward[i] and self.counter.compare(current.forward[i].value, value, "<"):
                current = current.forward[i]
            update[i] = current

        # Determina el nivel del nuevo nodo:
        lvl = self.random_level()
        if lvl > self.level:
            for i in range(self.level+1, lvl+1):
                update[i] = self.header
            self.level = lvl

        # Crea el nuevo nodo con el nivel generado:
        new_node = SkipNode(value, lvl)
        for i in range(lvl+1):
            new_node.forward[i] = update[i].forward[i]
            update[i].forward[i] = new_node

    # Convierte la Skip List en una lista de Python:
    def to_list(self):
        result = []
        current = self.header.forward[0]
        while current:
            result.append(current.value)
            current = current.forward[0]
        return result

# Funcin auxiliar para ordenar un arreglo usando Skip List:
def skiplist_sort(arr, counter):
    sl = SkipList()
    sl.counter = counter
    for v in arr:
        sl.insert(v)
    return sl.to_list()

Discusin

Los resultados obtenidos muestran diferencias notables en el desempeo de los algoritmos tanto en el nmero de comparaciones como en el tiempo de ejecucin. En trminos generales, los algoritmos con complejidad $O(n \log{n})$ (HeapSort, MergeSort y QuickSort) superaron ampliamente al algoritmo de tipo cuadrtico $O(n^2)$ (BubbleSort), mientras que la estructura SkipList mostr un comportamiento competitivo en ambos indicadores.

En la tabla de comparaciones, BubbleSort realiz de forma consistente el mayor nmero de comparaciones (4950 en todos los casos), lo que concuerda con su naturaleza iterativa y su baja eficiencia para listas medianamente grandes o con desorden. En contraste, MergeSort y HeapSort mantuvieron valores muy estables, alrededor de 5301040 comparaciones, evidenciando su eficiencia y su independencia relativa del nivel de perturbacin en los datos. QuickSort mostr mayor variabilidad y un nmero de comparaciones ms alto (entre 1900 y 2400), lo cual puede atribuirse a la seleccin del pivote y al desbalance en las particiones que este genera, especialmente cuando la lista no se distribuye de manera uniforme.

Por otro lado, la estructura SkipList alcanz un desempeo intermedio: su nmero de comparaciones fue cercano al de MergeSort, aunque con ligeras fluctuaciones segn el nivel de perturbacin (entre 795 y 961). Esto se debe a que la SkipList mantiene un orden probabilstico que depende del factor de promocin aleatoria en los niveles, lo cual puede afectar ligeramente la eficiencia entre ejecuciones o conjuntos con distinta distribucin.

En cuanto al tiempo de ejecucin, los resultados muestran coherencia con las comparaciones realizadas. MergeSort fue el algoritmo ms rpido en la mayora de los casos (0.00025 s), seguido por HeapSort y SkipList con tiempos promedio cercanos (0.00035 s). QuickSort present una ligera penalizacin temporal (0.001 s), probablemente debido a la sobrecarga en las operaciones de recursin y manejo de listas en Python. BubbleSort, aunque no mostr tiempos excesivos por el tamao reducido de los datos, fue el ms lento de los cinco (0.00115 s), reflejando su ineficiencia esperada.

En conjunto, los resultados confirman que MergeSort ofrece el mejor equilibrio entre eficiencia y estabilidad, seguido de HeapSort y SkipList como alternativas viables. QuickSort mantiene un buen rendimiento, pero su desempeo puede degradarse dependiendo de la naturaleza de los datos. Finalmente, BubbleSort, si bien til para propsitos didcticos, demuestra ser el menos adecuado para aplicaciones prcticas donde se requiera escalabilidad o rapidez.
