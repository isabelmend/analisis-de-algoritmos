Introduccin

La bsqueda de informacin dentro de estructuras de datos ordenadas constituye una operacin fundamental en el diseo y anlisis de algoritmos. En numerosos contextos (como bases de datos, sistemas de recuperacin de informacin, compiladores o motores de bsqueda) la eficiencia con que se localiza un elemento especfico determina directamente el rendimiento global del sistema (Cormen et al., 2022). Los algoritmos de bsqueda por comparacin representan una de las estrategias ms estudiadas y aplicadas, ya que permiten evaluar la eficiencia de diferentes mtodos en funcin del nmero de comparaciones realizadas y del tiempo requerido para localizar un elemento dentro de un conjunto de datos.

El principio general de estos algoritmos consiste en comparar iterativamente un elemento objetivo con los contenidos de una lista o estructura ordenada hasta encontrar una coincidencia o determinar su posicin. La diferencia esencial entre los mtodos radica en la forma de exploracin del espacio de bsqueda. Por ejemplo, la bsqueda secuencial $(B_0)$ recorre los elementos uno a uno, con una complejidad promedio de orden $O(n)$, adecuada nicamente para listas pequeas o sin orden especfico (Knuth, 1998). Las variantes no acotadas como $B_1$ y $B_2$ emplean saltos o expansiones exponenciales para delimitar intervalos, mejorando el rendimiento en algunos escenarios (Bentley & Yao, 1976).

Por otro lado, la bsqueda binaria utiliza la estructura ordenada de los datos, dividiendo el conjunto en mitades sucesivas y logrando una complejidad de $O(\log{n})$, que la convierte en una de las tcnicas ms eficientes para listas estticas (Cormen et al., 2022). En contraste, estructuras ms modernas como la Skip List introducen un enfoque probabilstico que permite mantener tiempos promedio de bsqueda, insercin y eliminacin tambin en $O(\log{n})$, con una implementacin ms simple (Pugh, 1990).

El objetivo de esta prctica es analizar y comparar experimentalmente distintos algoritmos de bsqueda por comparacin, evaluando su comportamiento en trminos del nmero total de comparaciones y del tiempo promedio de ejecucin al aplicarse sobre listas de diferente tamao y nivel de perturbacin. A travs de esta evaluacin, se busca contrastar el comportamiento experimental con la complejidad terica esperada y comprender las condiciones bajo las cuales cada algoritmo resulta ms eficiente. Este anlisis permite reforzar la comprensin de los principios de diseo algortmico y de la relacin entre costo computacional y estructura de datos.

class Comparador:
    def __init__(self):
        self.count = 0

    # Funcin que realiza una comparacin entre 'a' y 'b' utilizando el operador
    # indicado. Incrementa el contador cada vez que se ejecuta una comparacin.
    def compare(self, a, b, op):
        self.count += 1
        if op == "<":
            return a < b
        elif op == "<=":
            return a <= b
        elif op == "==":
            return a == b
        elif op == ">":
            return a > b
        elif op == ">=":
            return a >= b
        else:
            raise ValueError("Operador no permitido")

def busqueda_secuencial(lista, x, comp: Comparador):
    """
    Realiza una bsqueda secuencial (lineal) en una lista ordenada.

    Parmetros:
    lista : Lista ordenada de elementos donde se realizar la bsqueda.
    x : Elemento que se desea encontrar.
    comp : Objeto de tipo Comparador que define el mtodo compare(a, b, operador)
        para realizar las comparaciones.

    Retorna:
    True si el elemento se encuentra en la lista, False en caso contrario.
    """

    for v in lista:
        if comp.compare(v, x, "=="):
            return True
        elif comp.compare(v, x, ">"):
            break

    return False

def busqueda_binaria(lista, x, comp: Comparador):
    """
    Realiza una bsqueda binaria en una lista ordenada.Este algoritmo divide
    repetidamente la lista en mitades para reducir el espacio de bsqueda. En
    cada iteracin, compara el elemento del medio con el valor buscado.

    Parmetros:
    lista : Lista ordenada de elementos donde se realizar la bsqueda.
    x : Elemento que se desea encontrar.
    comp : Objeto de tipo Comparador que define el mtodo compare(a, b, operador)
        para realizar las comparaciones.

    Retorna:
    True si el elemento se encuentra en la lista, False en caso contrario.
    """

    # Se definen los lmites iniciales del rango de bsqueda:
    izq, der = 0, len(lista) - 1

    # Repetimos mientras haya un rango vlido:
    while izq <= der:
        mid = (izq + der) // 2

        # Comparar el elemento medio con el valor buscado:
        if comp.compare(lista[mid], x, "=="):
            return True

        # Si el valor medio es menor que 'x', se descarta la mitad izquierda:
        elif comp.compare(lista[mid], x, "<"):
            izq = mid + 1

        # Si el valor medio es mayor, se descarta la mitad derecha:
        else:
            der = mid - 1

    return False

def busqueda_no_acotada_B1(lista, elemento, counter):
    """
    Realiza una bsqueda no acotada B1. Se utiliza cuando no se conoce el tamao
    efectivo de la lista o cuando se desea reducir el rango de bsqueda
    antes de aplicar otro algoritmo.

    Proceso:
    1. Se comienza con un salto inicial de tamao 1.
    2. Mientras el elemento actual sea menor que el buscado,
       el salto se duplica, avanzando en la lista.
    3. Cuando se encuentra un elemento mayor o igual al buscado,
       se define un intervalo [i/2, i] como posible rango de bsqueda.
    4. Se realiza una bsqueda secuencial dentro de ese rango.

    Parmetros:
    lista : Lista ordenada en la que se realiza la bsqueda.
    elemento : Elemento que se desea encontrar.
    counter : Objeto con el mtodo compare usado para contar comparaciones.

    Retorna:
    True si el elemento se encuentra en la lista, False en caso contrario.
    """

    n = len(lista)
    salto = 1
    i = 0

    # Fase exponencial:
    while i < n and counter.compare(lista[i], elemento, "<"):
        i += salto
        salto *= 2

    # Determinar el rango acotado para bsqueda secuencial:
    inicio = max(0, i // 2)
    fin = min(i, n)

    # Bsqueda secuencial en el rango encontrado:
    for j in range(inicio, fin):
        if counter.compare(lista[j], elemento, "=="):
            return True

    return False

def busqueda_no_acotada_B2(lista, elemento, counter):
    """
    Realiza una bsqueda no acotada tipo B2 (bsqueda exponencial con bsqueda
    binaria final). Es til en listas grandes donde el tamao no es conocido o
    se accede de manera progresiva.

    Proceso:
    1. Se comienza verificando el primer elemento de la lista.
    2. Si el elemento buscado no est ah, se aumenta el ndice de manera
       exponencial hasta sobrepasar o igualar al elemento buscado.
    3. Una vez delimitado el intervalo [i/2, i], se aplica bsqueda binaria.
    4. Si el elemento se encuentra, se devuelve True; en caso contrario, False.

    Parmetros:
    lista : Lista ordenada en la que se realiza la bsqueda.
    elemento : Elemento que se desea encontrar.
    counter : Objeto con el mtodo compare usado para contabilizar comparaciones.

    Retorna: True si el elemento se encuentra en la lista, False en caso contrario.
    """

    n = len(lista)
    if n == 0:
        return False

    if counter.compare(lista[0], elemento, "=="):
        return True

    # Fase exponencial:
    i = 1
    while i < n and counter.compare(lista[i], elemento, "<"):
        i *= 2

    # Se definen los lmites del rango acotado:
    izquierda = i // 2
    derecha = min(i, n)

    # Fase de bsqueda binaria en el rango [izquierda, derecha):
    while izquierda < derecha:
        medio = (izquierda + derecha) // 2

        if counter.compare(lista[medio], elemento, "=="):
            return True
        elif counter.compare(lista[medio], elemento, "<"):
            izquierda = medio + 1
        else:
            derecha = medio

    return False

class Node:
    """
    Clase que representa un nodo dentro de la SkipList. Cada nodo almacena una
    clave y una lista de punteros que apuntan a otros nodos en distintos niveles.

    Atributos:
    key : Valor o clave almacenada en el nodo.
    forward : Lista de referencias a los siguientes nodos en cada nivel.
    """

    def __init__(self, key, level):
        self.key = key
        self.forward = [None] * (level + 1)


class SkipList:
    """
    Implementacin de una SkipList. Es una lista ordenada que permite operaciones
    de bsqueda, insercin y eliminacin gracias a su estructura multinivel
    basada en probabilidades.

    Parmetros:
    max_level : Nmero mximo de niveles que puede tener la lista.
    p : Probabilidad usada para determinar cuntos niveles tendr
        cada nuevo nodo.

    Atributos:
    header : Nodo cabecera (sin clave) que sirve como punto de partida.
    level : Nivel actual ms alto de la lista (crece dinmicamente).
    """

    def __init__(self, max_level=12, p=0.25):
        self.max_level = max_level
        self.p = p
        self.header = Node(None, self.max_level)
        self.level = 0

    def random_level(self):
        """
        Genera aleatoriamente un nivel para un nuevo nodo, de acuerdo con la
        probabilidad `p`. Este mtodo determina cuntos niveles tendr un nodo
        recin insertado.

        Retorna:
        lvl : Nivel aleatorio generado para el nuevo nodo.
        """
        lvl = 0
        while random.random() < self.p and lvl < self.max_level:
            lvl += 1
        return lvl

    def insert(self, key):
        """
        Inserta una nueva clave en la SkipList manteniendo el orden.
        """

        update = [None] * (self.max_level + 1)
        current = self.header

        # 1: Busca la posicin de insercin en cada nivel:
        for i in reversed(range(self.level + 1)):
            while current.forward[i] and current.forward[i].key < key:
                current = current.forward[i]
            update[i] = current

        # 2: Se posiciona en el siguiente nodo (nivel 0):
        current = current.forward[0]

        # 3: Inserta si la clave no est duplicada:
        if current is None or current.key != key:
            lvl = self.random_level()

            # Si el nuevo nivel excede el nivel actual, se actualiza la lista:
            if lvl > self.level:
                for i in range(self.level + 1, lvl + 1):
                    update[i] = self.header
                self.level = lvl

            # 4: Crea el nuevo nodo e inserta referencias:
            new_node = Node(key, lvl)
            for i in range(lvl + 1):
                new_node.forward[i] = update[i].forward[i]
                update[i].forward[i] = new_node

    def search(self, key, counter):
        """
        Busca una clave en la SkipList utilizando mltiples niveles.

        Proceso:
        1. Inicia desde el nodo cabecera en el nivel ms alto.
        2. Avanza hacia adelante mientras la clave actual sea menor que la buscada.
        3. Cuando no puede avanzar ms en ese nivel, desciende un nivel.
        4. Finalmente, verifica si la clave del nodo encontrado coincide.

        Parmetros:
        key : Clave que se desea buscar.
        counter : Objeto contador que registra el nmero de comparaciones.

        Retorna:
        True si la clave se encuentra, False en caso contrario.
        """

        current = self.header
        # Fase de navegacin descendente:
        for i in reversed(range(self.level + 1)):
            while current.forward[i] and counter.compare(current.forward[i].key,
                                                         key, "<"):
                current = current.forward[i]

        # Verificacin final en el nivel ms bajo:
        current = current.forward[0]
        if current and counter.compare(current.key, key, "=="):
            return True
        return False

def evaluar_busquedas(listas_posteo, consultas, path="."):
    """
    Evala el rendimiento de distintos algoritmos de bsqueda sobre mltiples
    listas de posteo y diferentes conjuntos de consultas. Calcula el nmero de
    comparaciones y el tiempo de ejecucin para cada algoritmo, mostrando los
    resultados totales y promedios.

    Parmetros
    listas_posteo : Archivos JSON con las listas de posteo a evaluar.
    consultas : Archivos JSON con las listas de consultas a buscar.
    path : Ruta del directorio donde se encuentran los archivos.

    Retorna
    df : Resultados de cada ejecucin (por lista, consulta y algoritmo).
    resumen_global : Promedios y totales agrupados por tipo de consulta y algoritmo.
    """

    resultados = []

    # Funcin auxiliar para convertir valores a enteros:
    def try_int(x):
        try:
            return int(x)
        except Exception:
            return x

    # Se recorre cada lista de posteo:
    for lista_file in listas_posteo:
        nombre_lista = lista_file.replace("listas-posteo-con-perturbaciones-", "").replace(".json", "")

        with open(f"{path}/{lista_file}", "r") as f:
            data_lista = json.load(f)

        # Unificar sublistas dentro del archivo en una sola lista plana:
        todos_elementos = []
        for key in data_lista:
            for v in data_lista[key]:
                todos_elementos.append(try_int(v))

        # Se verifican tipos de datos y se ordenan:
        tipos = set(type(x) for x in todos_elementos if x is not None)
        if len(tipos) == 0:
            print(f"Advertencia: lista {nombre_lista} vaca.")
            todos_elementos_sorted = []
        elif len(tipos) == 1:
            todos_elementos_sorted = sorted(todos_elementos)
        else:
            todos_elementos_sorted = sorted([str(x) for x in todos_elementos])

        #print(f"{nombre_lista}: elementos totales = {len(todos_elementos_sorted)}")

        # Se crea una estructura SkipList para este conjunto de datos:
        skiplist = SkipList()
        for v in todos_elementos_sorted:
            skiplist.insert(v)

        # Se evala cada conjunto de consultas sobre la lista actual:
        for consultas_file in consultas:
            nombre_consulta = consultas_file.replace("-listas-posteo.json", "")

            with open(f"{path}/{consultas_file}", "r") as f:
                consultas_data = json.load(f)

            if len(todos_elementos_sorted) == 0:
                continue

            sample_type = type(todos_elementos_sorted[0])

            # Se normalizan las consultas para que sean del mismo tipo:
            consultas_norm = []
            for q in consultas_data:
                q_conv = try_int(q)
                if sample_type is int and isinstance(q_conv, int):
                    consultas_norm.append(q_conv)
                else:
                    consultas_norm.append(str(q))

            #print(f"  {nombre_consulta}: consultas = {len(consultas_norm)}, tipo = {sample_type.__name__}")

            # Se definen los algoritmos a evaluar:
            algoritmos = {
                "B_0": busqueda_secuencial,
                "Binaria": busqueda_binaria,
                "B_1": busqueda_no_acotada_B1,
                "B_2": busqueda_no_acotada_B2,
                "SkipList": None
            }

            # Evaluacin de cada algoritmo
            for nombre, funcion in algoritmos.items():
                total_comps = 0
                inicio = time.time()

                for q in consultas_norm:
                    comp = Comparador()
                    if nombre == "SkipList":
                        skiplist.search(q, comp)
                    else:
                        funcion(todos_elementos_sorted, q, comp)
                    total_comps += comp.count

                tiempo_total = time.time() - inicio
                promedio = total_comps / len(consultas_norm) if consultas_norm else 0

                resultados.append({
                    "Query": nombre_consulta,
                    "List": nombre_lista,
                    "Algorithm": nombre,
                    "Tot_comparisons": total_comps,
                    "Avg_comparisons": round(promedio, 3),
                    "Tot_time(s)": round(tiempo_total, 6),
                    "Avg_time(s)": round(tiempo_total / len(consultas_norm) if consultas_norm else 0, 6)
                })

    # Crear DataFrames con resultados:
    df = pd.DataFrame(resultados)

    if df.empty:
        print("No se generaron resultados. Verificar archivos.")
        return None, None

    # Resumen global agrupado por consulta y algoritmo:
    resumen_global = (
        df.groupby(["Query", "Algorithm"])
          .agg({
              "Tot_comparisons": "sum",
              "Avg_comparisons": "mean",
              "Tot_time(s)": "sum",
              "Avg_time(s)": "mean"
          })
          .reset_index()
          .sort_values(["Query", "Algorithm"])
    )

    # Mostrar resultados:
    pd.set_option("display.max_rows", None)
    pd.set_option("display.max_columns", None)
    pd.set_option("display.width", 120)
    pd.set_option("display.colheader_justify", "center")
    pd.set_option("display.precision", 6)

    print("\n RESULTADOS DETALLADOS:")
    display(df.sort_values(["Query", "List", "Algorithm"]).reset_index(drop=True))

    print("\n RESUMEN GLOBAL (Totales y Promedios por Consulta):")
    display(resumen_global)

    return df, resumen_global

Los resultados obtenidos reflejan de manera coherente el comportamiento terico esperado de cada algoritmo de bsqueda en funcin del tamao de las listas y la complejidad de cada mtodo. En general, se observa una tendencia creciente en el nmero total de comparaciones y en el tiempo de ejecucin conforme aumenta el tamao de las listas de consulta, siendo esta relacin ms evidente en los algoritmos de bsqueda secuencial y ms controlada en los algoritmos con complejidad logartmica.

En primer lugar, los algoritmos $B_0$, $B_1$ y $B_2$, correspondientes a distintas variantes de bsqueda secuencial, muestran un crecimiento exponencial en las comparaciones totales al pasar del archivo consultas-1 al de consultas-4. Por ejemplo, $B_0$ alcanza ms de $1.18 \times 10^{10}$ comparaciones en consultas-4, lo cual se traduce en un tiempo total superior a 2200 segundos. Este comportamiento confirma su complejidad temporal $O(n)$, evidenciando que su rendimiento se degrada rpidamente conforme crece el tamao de las listas. Las versiones optimizadas ($B_1$ y $B_2$) presentan mejoras significativas al reducir el nmero promedio de comparaciones en cada conjunto de pruebas, aunque mantienen la misma tendencia de crecimiento, como se espera en los algoritmos lineales.

La bsqueda binaria mantiene un nmero promedio de comparaciones cercano a 28 en todas las consultas, con tiempos promedio del orden de $10^{-6}$ a $10^{-5}$ segundos, reflejando su complejidad terica $O(\log{n})$. En cuanto a la Skip List, su comportamiento fue notablemente competitivo respecto a la bsqueda binaria. En particular, el nmero de comparaciones promedio se mantuvo dentro de un rango similar (entre 13 y 30 comparaciones en promedio) y los tiempos de ejecucin fueron incluso menores en algunos escenarios. En consultas-3 y consultas-4, los valores promedio de comparaciones fueron de 25.7 y 29.8 respectivamente, lo cual demuestra que se logra mantener la eficiencia esperada incluso en grandes volmenes de datos. Estos resultados son coherentes con la teora propuesta por Pugh (1990), donde la Skip List combina la eficiencia promedio de una bsqueda binaria con la flexibilidad de las listas enlazadas, manteniendo un costo esperado de $O(\log{n})$ tanto en bsqueda como en insercin y eliminacin.

Conclusin

El anlisis comparativo de los diferentes algoritmos de bsqueda muestra que la eficiencia depende estrechamente del tipo de estrategia empleada y de la estructura de datos utilizada. Los mtodos de bsqueda lineal, como $B_0$ (secuencial) y $B_1$, presentan un crecimiento proporcional en el nmero de comparaciones y en el tiempo de ejecucin conforme aumenta el tamao de la lista, lo que limita su desempeo en escenarios de gran volumen de datos. En contraste, los algoritmos $B_2$ (exponencial), Binaria y SkipList mantienen un comportamiento estable y eficiente, reflejando una complejidad promedio de orden $O(\log{n})$.

Entre estos ltimos, la SkipList destaca por su balance entre simplicidad y rendimiento, ofreciendo tiempos y comparaciones similares o incluso inferiores a los de la bsqueda binaria, gracias a su estructura probabilstica que permite accesos rpidos sin requerir un arreglo contiguo. En conjunto, los resultados confirman que las estrategias basadas en bsqueda logartmica resultan significativamente ms adecuadas para el manejo eficiente de grandes volmenes de informacin, reafirmando la importancia de la seleccin del algoritmo de bsqueda en funcin del contexto de aplicacin y de la naturaleza de los datos.
